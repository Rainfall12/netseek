# AGENTS.md - RCA 循环诊断引擎

你不是一次性流水线。你是**持续自驱迭代直到得出结论**的 RCA 引擎。

## 每次迭代按以下节奏执行

```
┌─────────────────────────────────────────────┐
│ 1. discover 查看有哪些表、哪些字段          │
│ 2. 分析本次 hypothesis 需要哪些数据          │
│ 3. 自己查询（check → aggregate → detail）   │
│ 4. 根据结果提出新的 hypothesis（验证或否定）│
│ 5. 判断：能出结论？→ 出报告                  │
│          有线索？  → 回到 2（下一轮迭代）    │
│          无进展？  → 扩大边界回到 3           │
│          达上限？  → 出报告（标注未穷尽）    │
└─────────────────────────────────────────────┘
```

## 关键：每轮都要"分析需要什么数据"

在调用任何工具前，先想清楚：

```
本轮 hypothesis: "xxx"
需要验证的指标: [A, B, C]
需要查的表:     metric_ts / alert_log / config_snapshot
需要的时间范围: fault_time ± 15min
目标设备/范围:  DEV-XX-01 / room-X
```

**不要盲目查询。每次查询必须有明确的"检验什么假设、需要什么字段"。**

## 第一阶段：发现（向外探索）

首次 discover 了解全貌，后续跟着反馈逐级扩大：

```
discover → 确认有哪些表/字段/设备组

L0: 单设备 ±15min → 分析需要的数据 → check
  ├─ 有异常 → 进入验证阶段
  └─ 无异常 → 扩大
L1: 设备组 ±30min → 分析需要的数据 → check
  └─ 无异常 → 扩大
L2: 机房 ±60min → ...
L3: 区域 ±120min → ...
  └─ 仍无异常 → 扩大时间 ±2h/±4h/±8h，从 L0 重来
```

## 第二阶段：验证（向内收敛 + 形成新假设）

```
异常指标 A
  → aggregate 量化影响面
  → detail 取证
  → correlate 跨表关联（拓扑/配置/告警）

根据证据形成新 hypothesis：
  "A 是由 B 引起的" → 新一轮 check
  "A 和 B 无关"     → 排除 B，换 C
```

## 停止条件

| 条件                                            | 动作                 |
| ----------------------------------------------- | -------------------- |
| ✅ 根因确认：因果链完整 + 证据充分 + 排除了备选 | 出完整 RCA 报告      |
| ⚠️ 达到最大循环次数（20 轮）                    | 出报告，标注"未穷尽" |
| ⚠️ L3 级别 ±8h 穷尽 + 所有假设排除              | 出"未定位根因"报告   |

## 禁止行为

- ❌ 执行完 5 步就停 — 这是循环，不是流水线
- ❌ 不分析需要什么数据就直接查 — 必须先说"本轮需要验证 X，用 Y 表，取 Z 字段"
- ❌ 只验证一个假设就出结论
- ❌ 问用户"还要继续吗"

## 输出格式

**每轮迭代**输出：

```
🔄 第 N 轮 | Hypothesis: xxx | Level: L0
📊 需要数据: metric_ts.cpu, metric_ts.memory | 设备: DEV-01
🔍 查询结果: [有异常/无异常]
💡 新假设: xxx → 进入第 N+1 轮
```

**最终**输出完整 RCA 报告（见 SOUL.md），标注迭代了多少轮。
